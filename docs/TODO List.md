# Список задач

## Витрина промптов (супергруппа)

- Пользователь не зарегистрирован в боте. Нажимает кнопку скопировать промпт себе или открывает в редакторе и сохраняет оттуда. При сохранении Происходит регистрация пользователя в боте. В поле fsm_data добавляется пометка, что пользователь не заходил в личный чат с ботом. Для него записывеется выбранный промпт и выводится сообщение (решить как), где сохранен промпт и как туда попасть (ссылка на личный чат с ботом). Этои промпты будут выведены в чат, когда пользователь стартанет его.


Пишем новую функцию в отдельном файле js.

Для получения  шаблонов и пецификации вызовав ознакомся с документом @/docs/frontend/templates_usage.md , он описывает скрипт в файле @/services/web_app/static/templates.js (этот файл не менять, использовать для ознакомления).

Функция будет принимать путь к элементу в общем шаблоне (шаблон находит в глобальном хранилище appData, он задается в @/services/web_app/static/start.js ), контент, который нужно вставить в создаваемый элемент, если это требуется и datа-id, если не передан, он равен 1 (используется для индексации элементов массива).

Читает элемент по указанному пути из общего шаблона формы. Определяет его тип, получает данные этого элемента (в том числе  datа-id, если надо) и использует их для подготовки набора заполнителей плейсхолжеров и определения какой шаблон нужно использовать.

Создавай все в отдельных функциях, по одной на группу).

Полученный код вернуть из функции в отформатированном виде.
