# Справочник по проекту JPrompter

Этот документ — ваш путеводитель по всей документации проекта. Он поможет вам быстро найти нужную информацию о бэкенде, фронтенде и общих концепциях.

## 1. Общая архитектура и концепции

Здесь собрана информация о высокоуровневом устройстве проекта, безопасности и планах на будущее.

- **[Безопасность и авторизация (`app_security.md`)](app_security.md)**: Детальное описание процесса авторизации пользователя Telegram Mini App. Описывает, как происходит валидация данных, полученных от Telegram, и генерация JWT-токена для сессии. **Обязательно к прочтению** для понимания механизма аутентификации.

- **[База данных (`database.md`)](database.md)**: Описание структуры базы данных. Включает схемы таблиц `user`, `prompt_chat` и `prompt_group`, а также структуру JSON-поля `message`, в котором хранятся данные о промптах.

- **[Планы на будущее (`future.md`)](future.md)**: Список идей и доработок, которые не вошли в первую версию, но могут быть реализованы в будущем.

- **[Список задач (`TODO List.md`)](TODO%20List.md)**: Конкретные задачи, которые находятся в работе или запланированы.

## 2. Фронтенд (Telegram Mini App)

Этот раздел посвящен всему, что связано с клиентской частью приложения — интерфейсом редактора промптов.

### 2.1. Пользовательский интерфейс и логика

- **[Общее описание интерфейса (`frontend/interface.md`)](frontend/interface.md)**: Описывает три основных режима работы редактора (Формат, Текст, Медиа), логику кнопок на панели инструментов (Переключить режим, Форматировать, Сохранить и т.д.) и особенности поведения приложения при запуске из разных контекстов (личный чат, группа).

- **[Работа с формой и кнопками (`frontend/form_work.md`)](frontend/form_work.md)**: Техническое описание логики работы кнопок в динамической форме. Здесь объясняется, как работают действия с массивами (добавление, очистка) и объектами (клонирование, перемещение, удаление). **Ключевой документ** для понимания интерактивности формы.

### 2.2. Динамическое построение формы

Система генерации формы — ядро фронтенда. Она рекурсивно строит HTML на основе JSON-шаблона.

- **[Обзор системы построения форм (`frontend/create_form/create_form.md`)](frontend/create_form/create_form.md)**: Высокоуровневый обзор архитектуры, состоящей из трех компонентов: `createForm` (высокий уровень), `createElement` (средний уровень) и `renderTemplate` (низкий уровень). Описывает, как они взаимодействуют для построения всей формы.

- **[Спецификация `createElement` (`frontend/create_form/create_element_usage.md`)](frontend/create_form/create_element_usage.md)**: Документация по функции среднего уровня, которая отвечает за создание одного HTML-элемента (простого или контейнерного).

- **[Спецификация `renderTemplate` (`frontend/create_form/templates_usage.md`)](frontend/create_form/templates_usage.md)**: Документация по низкоуровневому шаблонизатору, который непосредственно генерирует HTML-строки для каждого типа поля (text, select, box и т.д.).

- **[Спецификация JSON-шаблона (`frontend/create_form/example_json_sample.md`)](frontend/create_form/example_json_sample.md)**: **Очень важный документ.** Описывает структуру JSON-шаблона, который используется для генерации формы. Здесь вы найдете описание всех доступных типов полей (`_type`), их атрибутов (`_label`, `_default`, `_options`) и то, как они преобразуются в HTML и итоговый JSON-объект.
